# Pricing Improvements — UI Guideline Spec Packs

> Fuente: Feedback de Shahab (email post-launch, Feb 19 2026)

## Accionables

### 1. Free Sample de componente complejo
- Regalar un spec pack de un componente no-trivial con ARIA real y keyboard interactions
- Candidatos: Select, Menu, Tabs
- Objetivo: reducir barrera de confianza — que prueben antes de comprar
- El usuario necesita ver que el spec pack realmente mejora el output de la IA, no solo "mejores nombres de props"

### 2. Upgrade Path ($9 → Full Library)
- Que la compra individual de $9 se descuente si después compra la full library
- Elimina la fricción psicológica: "si compro uno y me gusta, la library me costó $160 en vez de $149"
- Convierte el $9 en un trial sin riesgo

### 3. Aclarar qué incluye el tier de $149
- ¿Los componentes Q2 ("in progress") están incluidos o no?
- ¿Es realmente "full library" o es "lo que está listo ahora"?
- Propuesta: bugfixes/mejoras a lo comprado → incluidos. Componentes nuevos → no incluidos (o sí con el tier de $249)
- Considerar: $149 con 1 año de updates (mejoras + nuevos) podría ser el sweet spot

---

## Contexto
- Shahab es dev solo en startup pequeña, paga de su bolsillo
- $149 = impulso. $249 = "espero Black Friday"
- ES el usuario objetivo: personaliza librerías (React ARIA, MUI, PrimeVue, Headless UI), quiere ir from-scratch pero la IA sola no alcanza
